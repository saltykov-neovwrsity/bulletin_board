# Дошка оголошень (Bulletin Board)

Сучасний та швидкий веб-додаток для публікації та пошуку оголошень. Проєкт дозволяє користувачам додавати оголошення в різних категоріях (продаж, послуги, робота, інше), здійснювати пошук за ключовими словами та сортувати результати.

## Основні можливості

- **Перегляд оголошень:** Зручний інтерфейс у вигляді карток із відображенням категорії, ціни та опису.
- **Пошук та фільтрація:** Можливість швидко знайти потрібне оголошення за текстом.
- **Сортування:** Відображення від нових до старих і навпаки.
- **Створення оголошення:** Форма додавання з валідацією полів на стороні сервера.
- **Захищене видалення:** Видалення оголошень можливе лише після введення 4-значного PIN-коду, заданого при створенні.
- **Адаптивний дизайн:** Коректне відображення як на ПК, так і на мобільних пристроях.

## Технологічний стек

- **Backend:** Node.js, Express.js
- **Шаблонізатор:** EJS (Embedded JavaScript)
- **База даних:** SQLite
- **ORM:** Prisma
- **Стилізація:** Vanilla CSS (CSS Variables, Flexbox, Grid)

## Встановлення та запуск

Слідуйте цим крокам, щоб запустити проєкт локально:

### 1. Клонування репозиторію
```bash
git clone https://github.com/saltykov-neovwrsity/bulletin_board.git
cd bulletin_board
```

### 2. Встановлення залежностей
```bash
npm install
```

### 3. Налаштування бази даних (Prisma)
У проєкті використовується SQLite, тому додаткового налаштування сервера БД не потрібно. Просто виконайте міграції:
```bash
npx prisma migrate dev --name init
```

### 4. Запуск сервера
```bash
npm start
```
Сервер буде запущено за адресою: [http://localhost:3000](http://localhost:3000)

## Валідація даних
При створенні оголошення діють наступні правила:
- **Назва:** від 5 до 100 символів.
- **Опис:** мінімум 10 символів.
- **Категорія:** обов'язковий вибір з переліку (Продаж, Послуги, Робота, Інше).
- **Ціна:** тільки додатні числа більші за 0.
- **Контакти:** мінімум 5 символів.
- **PIN-код:** рівно 4 цифри (для захисту від несанкціонованого видалення).

## Адміністрування
Для видалення будь-якого оголошення (наприклад, для модерації або якщо користувач забув свій PIN), адміністратор може використати Майстер PIN-код. Для цього необхідно додати змінну у файл `.env`:
```env
MASTER_PIN=7777
```
